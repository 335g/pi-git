## 実装サマリー

### 変更ファイル

**`src/core/auto-commit-confirm.ts`**:
- `visibleWidth`, `truncateToWidth` を pi-tui からインポート追加
- `ConfirmComponentFactory` の `tui` パラメータ型を拡張（`terminal?: { rows: number }` 追加）
- `MAX_RENDERED_LINES = 200` 定数追加（極端に広い端末の安全キャップ）
- コンストラクタで `this.termHeight` を `tui.terminal.rows` から取得（フォールバック 24）
- `render()` を全画面対応に書き換え:
  - コンテンツ行を先に構築（既存ロジック維持）
  - `theme.bg("selectedBg")` で各行に背景色を適用
  - `visibleWidth()` でANSIコードを考慮した水平方向センタリング
  - `truncateToWidth()` でファイル名切り詰め（ANSIコード・マルチバイト対応）
  - `termHeight` を使って動的に垂直方向中央配置
  - 端末が小さい場合は padding=0 でコンテンツを先頭に
- `handleInput()` に Ctrl+C → No のハンドリング追加

**`src/core/auto-commit.ts`**:
- `OverlayOptions` 型を pi-tui からインポート追加
- `showConfirmDialog()` にフルスクリーン overlayOptions 追加:
  - `width: "100%"`, `maxHeight: "100%"`, `row: 0`, `col: 0`, `margin: 0`
  - overlay が端末全体を覆うように設定
