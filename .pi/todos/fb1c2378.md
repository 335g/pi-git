## 場所
`src/commands/agg-commit.ts:139-151`

## 問題
ある hunk のコミット失敗後、`resetStaging` が失敗すると（`catch` で握り潰し）、次の hunk の `stageFiles` が前の失敗分のファイルも含めてステージングしてしまう。結果として、誤ったスコープのコミットが作成される。

## 修正案
各 hunk のループ開始時に `resetStaging` を実行してクリーンスレートを保証する：
```typescript
await resetStaging(pi, ctx.cwd);  // clean slate before each hunk
await stageFiles(pi, hunk.files, ctx.cwd);
```
もしくは `resetStaging` 失敗時にバッチ全体をアボートする。
