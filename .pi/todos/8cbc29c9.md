## 場所
`src/core/auto-commit.ts:59`

## 問題
```ts
await footerManager.refresh();                     // L42
if (!(await isGitRepository(pi))) { return; }      // L48
if (!(await hasChanges(pi))) { return; }           // L52
await footerManager.setRunning("auto-commit", ...); // L59  ← try の外

try {
    // commit logic
} finally {
    await footerManager.clearRunning();
}
```
`setRunning()` (L59) が失敗すると `clearRunning()` が呼ばれずフッターが固化する。`agent_end` ハンドラーの try/catch では Promise Rejection は捕捉されるが、状態バグは防げない。

なお L42 `refresh()`, L48 `isGitRepository()`, L52 `hasChanges()` も try 外だが、これらは L17-22 の早期リターンガードであり失敗時の副作用は小さい。

## 修正案
`setRunning` を `try` ブロック内に移動する。
